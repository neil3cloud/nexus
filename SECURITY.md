# Security Policy

## Scope

Security concerns include vulnerabilities in code, unsafe defaults, accidental exposure of sensitive project data, and design flaws that could undermine trust in Shared Reality.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately to the maintainers rather than opening a public issue.

## Handling Expectations

- Reports will be acknowledged as capacity allows.
- Valid issues will be triaged based on severity and architectural impact.
- Security-related design changes should be captured in the Engineering Corpus when appropriate.

## Secure Development Guidance

- Avoid embedding secrets in source or documentation.
- Preserve Adapter independence and least-privilege boundaries.
- Prefer explainable, auditable flows over opaque automation.
