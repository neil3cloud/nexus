# Contributing to Nexus

Thank you for helping shape Nexus.

## Contribution Principles

- Protect the architectural invariants in `/knowledge/architecture`.
- Prefer clear contracts and documentation before implementation.
- Keep hosts thin and business logic inside platform layers.
- Add or update ADRs when architectural direction changes.
- Strengthen the Engineering Corpus with every meaningful change.

## Expected Contribution Flow

1. Read `README.md` and `ARCHITECTURE.md`.
2. Review relevant corpus documents in `/knowledge`.
3. Open or reference an issue describing the change.
4. Add or update architectural documentation when needed.
5. Keep changes focused, explainable, and reversible.

## Documentation Expectations

New architecture-facing work should reference related corpus artifacts and explain how it affects Shared Reality, capabilities, or Adapter boundaries.

## Code Expectations

The current repository contains contracts and placeholders only. Future implementations should be incremental, well-scoped, and aligned with documented architectural decisions.
