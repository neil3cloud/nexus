# Roadmap

## Purpose

Describe intended product evolution without locking the architecture into premature implementation choices.

## Responsibilities

- Track strategic themes
- Connect near-term steps to long-term direction
- Make evolution visible to contributors

## Scope

Roadmap artifacts should reflect architectural intent, not speculative promises.

## Future Evolution

This area can later evolve into milestone planning, adoption sequencing, and workflow maturity maps.

## Relationship to Shared Reality

The roadmap should show how Shared Reality and repository knowledge will become stronger over time.
