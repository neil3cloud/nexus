## Normative Source of Truth

Behavioral rules for Shared Reality are governed by the Kernel Canon and RFC-0003.

## Definition

Shared Reality is the continuously evolving, evidence-backed understanding of a software project that Nexus maintains for humans and AI systems.

## Sources of Reality

Shared Reality is assembled from source code, architecture documents, ADRs, repository policies, git state, diagnostics, active mission context, accepted implementation history, and other validated engineering evidence.

## Architectural Consequences

- Engineering workflow must begin with evidence assembly.
- Recommendations must be explainable.
- Project knowledge should improve future sessions without turning into an autonomous memory system.
