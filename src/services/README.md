# Services

## Purpose

Reserve application services that compose lower-level contracts without becoming hosts.

## Responsibilities

- Coordinate reusable flows
- Encapsulate cross-module operations
- Remain aligned with kernel boundaries

## Scope

Services should remain thin and policy-aware.

## Future Evolution

Future work may introduce focused services for knowledge capture, planning support, or evidence retrieval.

## Relationship to Shared Reality

Services can help operationalize Shared Reality while preserving kernel service boundaries.
