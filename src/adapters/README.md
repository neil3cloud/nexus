# Adapters

## Purpose
Describe boundary components that integrate Nexus with external systems.

## Responsibilities
- Translate platform contracts to external APIs
- Isolate implementation-specific concerns
- Preserve kernel purity

## Scope
Adapters should not redefine platform behavior or architectural concepts.

## Future Evolution
Future adapters may target source control, issue systems, model APIs, and storage backends.

## Relationship to the Engineering Corpus
Adapters let Shared Reality interact with external systems without collapsing architectural boundaries.
