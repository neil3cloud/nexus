## BT-074-001 — Add direct aggregate-level unit tests for the `Submitted → Under Review` transition and its rejection from `Draft`/`Withdrawn`

**Status:** OPEN

**Summary:**

`ProposedMissionPlan.markCurrentRevisionUnderReview` and `ProposedPlanRevision.transitionTo('Under Review')` (via the `isAllowedTransition` pair added by `NEXUS-RAT-2026-07-17-013`) are exercised only indirectly, through `PlanningCorrelationService.enterReview`'s integration-style tests in `test/kernel/planning/planning-correlation.test.ts`. `enterReview` independently asserts the current revision is `Submitted` (`assertSubmittedCurrentRevision`) *before* calling `markCurrentRevisionUnderReview`, so no existing test ever reaches the aggregate with a `Draft` or `Withdrawn` revision to verify `transitionTo` itself still rejects those transitions into `Under Review`.

**Authority:**

IMPLEMENTATION_CONSTITUTION.md — Testing ("Tests SHALL validate: aggregate behavior... state transitions"); `NEXUS-RAT-2026-07-17-013` ("reachable only from `Submitted`, rejected from `Draft` or `Withdrawn`"); Sprint 74 Implementation Record — Acceptance Criteria ("every rejection condition above is enforced and tested").

**References:**

- Originating finding: `NEXUS-REV-2026-07-17-014-F-001` (Category 1 — Implementation Defect, Minor).
- Sprint: Sprint 74 — Planning Correlation and Review Entry Foundation.
- RFC: RFC-0012 v1.0 — Autonomous Engineering Planning Model (Referenced; Proposal Lifecycle section).
- Ratification: `NEXUS-RAT-2026-07-17-013` (additive `Under Review` lifecycle-extension authorization).

**Implementation Targets:**

- `test/kernel/planning/planning-domain.test.ts` (or `test/kernel/planning/planning-correlation.test.ts`) — add new test cases only.
- No production source file is expected to change to satisfy this task.

**Required Changes:**

- Add a direct unit test constructing a `ProposedMissionPlan`/`ProposedPlanRevision` with a `Submitted` current revision and invoking `markCurrentRevisionUnderReview`/`transitionTo('Under Review')` directly (bypassing `PlanningCorrelationService`), asserting the resulting revision's `lifecycleState` is `'Under Review'`.
- Add a direct unit test invoking the same aggregate-level transition from a `Draft` current revision, asserting rejection with `InvalidProposalLifecycleTransitionError`.
- Add a direct unit test invoking the same aggregate-level transition from a `Withdrawn` current revision, asserting rejection with `InvalidProposalLifecycleTransitionError`.

**Acceptance Criteria:**

- No existing test is modified or removed.
- No production source file is modified (this is a test-coverage-only task; `NEXUS-REV-2026-07-17-014` recorded no production defect).

**Reviewer Evidence:**
