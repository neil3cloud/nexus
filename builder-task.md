# Future Improvements

## `NEXUS-REV-2026-07-17-005` Observation O-001 — Redundant, unreachable `processedEventIds` duplicate-check

Category 6, Observation, Informational (Sprint 68; new). `GovernanceGatedWorkflowAdvancementConsumer.handleEvent` maintains both a `resultsByEventId` cache (checked first, on every call, for any previously-seen `eventId`) and a separate `processedEventIds` Set (checked later, only reached after the `resultsByEventId` check has already passed). Every code path that adds to `processedEventIds` also calls `recordResult`, which populates `resultsByEventId` for the same `eventId` in the same call, so on any actual re-delivery of a previously-handled event the earlier `resultsByEventId` check always returns first — the `processedEventIds.has(...)` branch is provably unreachable. Not a correctness defect: the idempotency guarantee it exists to provide is already fully satisfied by the `resultsByEventId` cache, confirmed by test `S68-6`. No Builder Task. A future Sprint touching this consumer MAY simplify by removing the redundant `processedEventIds` Set.

## `NEXUS-REV-2026-07-17-003` Observation O-001 — Test name overstates Workflow-position lookup coverage

Category 6, Observation, Informational (Sprint 67; carried forward). Test `S67-7` ("resolves by Workflow position and fails closed on ambiguous repository state") is named as if it exercises Workflow-position lookup ambiguity, but its two constructed correlations use different `engineeringSessionId` values, so their position keys never collide; the ambiguity it actually exercises and asserts is `findByReviewId` ambiguity. The Reviewer confirmed by source inspection of `InMemoryEngineeringDecisionCorrelationRepository.register` that genuine position-key ambiguity is structurally unreachable through the public repository contract as implemented. Defensible, RFC-0004-consistent design choice, not a defect. No Builder Task. Not touched by Sprint 68.
