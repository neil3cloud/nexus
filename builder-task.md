# Future Improvements

## `NEXUS-REV-2026-07-17-003` Observation O-001 — Test name overstates Workflow-position lookup coverage

Category 6, Observation, Informational (Sprint 67; new). Test `S67-7` ("resolves by Workflow position and fails closed on ambiguous repository state") is named as if it exercises Workflow-position lookup ambiguity, but its two constructed correlations use different `engineeringSessionId` values, so their position keys never collide; the ambiguity it actually exercises and asserts is `findByReviewId` ambiguity. The Reviewer confirmed by source inspection of `InMemoryEngineeringDecisionCorrelationRepository.register` that genuine position-key ambiguity is structurally unreachable through the public repository contract as implemented: `register` returns the existing correlation whenever a position key already resolves to one, so the `correlationIdsByPositionKey` index can never index more than one correlation per key. This is a defensible, RFC-0004-v1.14-consistent design choice, not a defect. No Builder Task. A future Sprint touching this repository should keep this invariant in mind if position-key uniqueness is ever relaxed.

## `NEXUS-REV-2026-07-16-016` Observation 1 — `GovernanceStateProjectionService` replay-on-read is redundant with live subscription

Category 6, Observation, Informational (Sprint 63; carried forward). `getGovernanceStateProjection` maintains a live `EventBus` subscription and also unconditionally replays the Mission's entire event history on every retrieval call, re-applying already-consumed events. Harmless — dedupes by `eventId`, verified idempotent — but redundant for a continuously-subscribed Mission, and the per-event `includes()` dedup check is `O(n)`, making full replay-and-reapply `O(n²)` in the Mission's total event count. Immaterial at current scale. No Builder Task. Not touched by Sprint 65, 66, or 67.
