## BT-075-003 — Add a direct unit test for `assertExistingGovernanceDecisionCanBeSuperseded`'s terminal-outcome rejection branch

**Status:** OPEN

**Summary:**

`PlanningCorrelationService.assertExistingGovernanceDecisionCanBeSuperseded` resolves the currently-recorded `governanceDecisionId`'s `GovernanceDecision` via `governanceDecisionRepository.getById` and throws `PlanningCorrelationAssociationRejectedError` if that decision's outcome is terminal (`Approved` or `Rejected`). This branch is reachable — the `PlanningCorrelation` save (recording a `GovernanceDecision` reference) and the corresponding `ProposedPlanRevision` transition are two separate, non-atomic writes in `evaluateGovernance`; a failure between them followed by a retry attempting to record a *different* `GovernanceDecision` would exercise it — but no existing test constructs this scenario.

**Authority:**

IMPLEMENTATION_CONSTITUTION.md — Testing ("Tests SHALL validate: aggregate behavior... state transitions"); `NEXUS-RAT-2026-07-17-015` ("`governanceDecisionId` SHALL be immutable only once the referenced `GovernanceDecision` outcome is `Approved` or `Rejected`").

**References:**

- Originating finding: `NEXUS-REV-2026-07-17-018-F-001` (Category 1 — Implementation Defect, Minor).
- Sprint: Sprint 75 — Proposal Governance Integration.
- Ratification: `NEXUS-RAT-2026-07-17-015` (corrective scope this branch enforces).
- RFC: RFC-0012 v1.1 — Autonomous Engineering Planning Model (Referenced; Planning Correlation section).

**Implementation Targets:**

- `test/kernel/planning/planning-correlation.test.ts` — add a new test case only.
- No production source file is expected to change to satisfy this task.

**Required Changes:**

- Add a direct unit test that constructs (or drives, via the existing test harness's `governanceDecisionRepository` and `planningCorrelationRepository`) a `PlanningCorrelation` whose recorded `governanceDecisionId` already references a terminal (`Approved` or `Rejected`) `GovernanceDecision` while its associated `ProposedPlanRevision` remains `Under Review` — reproducing the non-atomic-write edge case the branch defends against.
- Assert that calling `evaluateGovernance` (or directly exercising `assertExistingGovernanceDecisionCanBeSuperseded` if a lower-level seam is more direct) with a *different* `governanceDecisionId` in that state is rejected with `PlanningCorrelationAssociationRejectedError`.

**Acceptance Criteria:**

- No production source file is modified (this is a test-coverage-only task; `NEXUS-REV-2026-07-17-018` recorded no production defect — the logic is correct as written).
