## TASK-001 — Correct the Host/Adapter drift self-check's git pathspec to forward-slash separators

Status

OPEN

### Category

Implementation Defect

### Severity

Minor

### Source

`NEXUS-REV-2026-07-16-013-F-001`

### Summary

The Sprint 62 integration test `'leaves existing Sprint 1-61 production contracts and Kernel boundaries unmodified'` (`test/integration/governance-automation-integration-validation.integration.test.ts`) checks for `src/hosts`/`src/adapters` drift using the backslash-separated pathspec literals `'src\\hosts'`, `'src\\adapters'`. Git for Windows normalizes backslashes to path separators in pathspecs, so the check passes locally on Windows, but POSIX git — including this repository's own `ubuntu-latest` CI runner (`.github/workflows/repository-foundation.yml`) — treats a backslash inside a pathspec as an escape character, not a directory separator. On that platform the check matches nothing and silently reports zero changed Host/Adapter files regardless of what actually changed.

### Authority

- Sprint 62's Sprint Implementation Record, Required Test Matrix item 3.
